# LTCPA 产品需求文档 (PRD)

> Product Requirements Document — 立堅會計師事務所官方網站
> 版本: v2.0 | 更新日期: 2026-04-30

---

## 1. 项目概述

### 1.1 项目背景

立堅會計師事務所（LTCPA）需要一個現代化、多語言的官方網站，用於展示服務、公司資訊、團隊介紹，並接收客戶查詢。網站需支持內容管理員通過 Web 界面自行更新內容，無需開發者介入。

### 1.2 项目目標

| 目標 | 說明 | 優先級 |
|------|------|--------|
| 品牌展示 | 專業、可信的會計師事務所形象 | P0 |
| 多語言支持 | 繁體中文、簡體中文、英文三語言 | P0 |
| 內容自助管理 | 非技術人員可通過 Admin 面板更新內容 | P0 |
| 客戶查詢收集 | 聯絡表單收集潛在客戶資訊 | P0 |
| 即時通知 | 表單提交後自動通知管理員（WhatsApp + Email） | P1 |
| 訪問分析 | 追蹤網站流量和用戶行為 | P1 |

### 1.3 目標用戶

| 用戶類型 | 需求 | 使用場景 |
|----------|------|----------|
| 潛在客戶 | 了解服務、提交查詢 | 瀏覽服務頁面、填寫聯絡表單 |
| 內容管理員 | 更新網站內容 | 登入 Admin 面板修改文字/圖片 |
| 管理層 | 查看網站數據 | 查看 Analytics 報表 |

## 2. 功能需求

### 2.1 網站頁面

| 頁面 | 說明 | 語言 |
|------|------|------|
| 首頁 (Home) | Hero 輪播、服務卡片、統計數字、Why Choose | 3 語言 |
| 關於我們 (About) | 公司簡介、團隊介紹、核心價值 | 3 語言 |
| 服務總覽 (Services) | 6 大服務類別概覽 | 3 語言 |
| 服務詳情 (Services/[slug]) | 審計、稅務、風險、法證、諮詢、交易 | 3 語言 |
| 聯絡我們 (Contact) | 表單、地址、辦公時間、WhatsApp | 3 語言 |
| Footer | 版權、社交鏈接、快速連結、WhatsApp | 3 語言 |

> 總計: 13 個頁面家族 × 3 語言 = 44 個靜態頁面

### 2.2 CMS 內容管理系統

#### 2.2.1 可編輯區域

| 區域 | 字段類型 | 說明 |
|------|----------|------|
| Site Settings | 文字、圖片 | 網站名稱、Logo、WhatsApp 號碼 |
| Navigation | 數組（文字+鏈接） | 主選單項目 |
| Home | 多區塊 | Hero、服務卡片、統計、Why Choose |
| About | 多區塊 | 簡介、團隊、價值 |
| Contact | 多區塊 | 表單文案、地址、電話、WhatsApp |
| Footer | 多區塊 | 版權、社交、連結 |
| Services Overview | 數組 | 6 項服務卡片 |
| Service Details | 嵌套數組 | 每項服務的 Scope Items（可增刪） |

#### 2.2.2 圖片管理

| 功能 | 需求 |
|------|------|
| 上傳 | 支持 jpg/png/webp，單張 < 10MB |
| 刪除 | 僅可刪除 R2 上傳的圖片，Static 圖片只讀 |
| 瀏覽 | All / Static / Uploaded 三標籤切換 |
| 引用 | 點擊複製 URL，填入內容字段 |

### 2.3 聯絡表單

| 字段 | 必填 | 驗證規則 |
|------|------|----------|
| 姓名 (Name) | ✅ | 非空 |
| 公司 (Company) | ✅ | 非空 |
| 電郵 (Email) | ✅ | Email 格式 |
| 電話 (Phone) | ❌ | - |
| 服務類型 (Service) | ❌ | 下拉選單（6 項 + 其他） |
| 訊息 (Message) | ❌ | - |

**提交後行為:**
1. 保存到 D1 數據庫
2. 發送 WhatsApp 通知到管理員
3. 發送 Email 通知到管理員（可選）
4. 前端顯示成功提示

### 2.4 WhatsApp 集成

| 功能 | 說明 |
|------|------|
| 網站顯示 | Footer 和 Contact 頁面顯示 WhatsApp 圖標和號碼 |
| 點擊跳轉 | 點擊後打開 `wa.me/<號碼>` |
| 表單通知 | 新查詢自動發 WhatsApp 通知到管理員 |
| 號碼管理 | 通過 Admin → Site Settings 修改，無需改代碼 |

### 2.5 分析統計 (Analytics)

| 指標 | 數據源 |
|------|--------|
| 總瀏覽量 | Pageview 事件計數 |
| 獨立訪客 | Session ID 去重 |
| 熱門頁面 | 按頁面路徑聚合 |
| 國家分布 | Cloudflare CF-IPCountry |
| 最近訪問 | 原始 pageview 記錄列表 |

### 2.6 站內搜索

| 功能 | 說明 |
|------|------|
| 觸發方式 | Ctrl+K 快捷鍵 |
| 搜索範圍 | 頁面正文內容（排除 header/nav/footer/script/style） |
| 結果高亮 | `<mark>` 標籤高亮匹配文本 |
| 導航 | 上一個/下一個匹配項 |

## 3. 非功能需求

### 3.1 性能

| 指標 | 目標 |
|------|------|
| TTFB | < 100ms（Cloudflare CDN） |
| FCP | < 1.5s |
| 頁面大小 | < 500KB（不含圖片） |
| 構建時間 | < 2 分鐘 |

### 3.2 可用性

| 指標 | 目標 |
|------|------|
| 正常運行時間 | > 99.9%（Cloudflare SLA） |
| 瀏覽器兼容 | Chrome, Safari, Firefox, Edge 最新 2 個版本 |
| 移動端適配 | 響應式設計，支持 320px+ |

### 3.3 安全

| 需求 | 實現 |
|------|------|
| XSS 防護 | 輸出轉義 + DOMPurify |
| SQL 注入 | D1 參數化查詢 |
| 認證 | Admin Token + 密碼驗證 |
| HTTPS | 強制 TLS（Cloudflare 自動） |
| CORS | 嚴格限制來源域名 |

### 3.4 可維護性

| 需求 | 實現 |
|------|------|
| 代碼組織 | 模塊化組件，清晰的目錄結構 |
| 類型安全 | TypeScript 全覆蓋 |
| 文檔 | AGENTS.md + docs/ 完整文檔 |
| 版本控制 | Git + GitHub Actions CI/CD |

## 4. 技術約束

| 約束 | 說明 |
|------|------|
| 靜態導出 | Next.js `output: 'export'`，無 SSR |
| 零託管成本 | 使用 Cloudflare 免費層 |
| 單文件 Admin | Admin 面板為單一 HTML 文件，不依賴 Next.js 運行時 |
| KV 最終一致性 | 保存後有短暫延遲（通常 < 60s） |

## 5. 驗收標準

### 5.1 功能驗收

- [ ] 所有 44 個頁面可正常訪問，無 404
- [ ] 三語言切換正常，內容對應正確
- [ ] Admin 面板可登入、編輯、保存、部署
- [ ] 聯絡表單提交後 D1 有記錄
- [ ] WhatsApp 通知成功發送
- [ ] Analytics 數據正常收集

### 5.2 性能驗收

- [ ] Lighthouse Performance > 90
- [ ] 所有頁面加載 < 3s（4G 網絡）
- [ ] 圖片延遲加載正常工作

### 5.3 安全驗收

- [ ] Admin 密碼錯誤無法登入
- [ ] 未授權無法調用 CMS API
- [ ] 表單 XSS 攻擊無效
